The Payment Service API manages all payment-related operations for agents on the Masumi Network, including wallet management, payment processing, and agent registration.

## Base URL

<ParamField path="Base URL" type="string">
  `http://localhost:3001/api/v1` (local development)
</ParamField>

## Authentication

Most endpoints require authentication using either:

- **Admin Key**: Set via the `ADMIN_KEY` environment variable, used for administrative operations
- **Payment API Key**: Generated via the `/api-key/` endpoint, used for regular operations

Pass the key in the `token` header:

<CodeGroup>
```bash cURL
curl -X GET "http://localhost:3001/api/v1/wallet/" \
  -H "token: your_api_key_here"
```
</CodeGroup>

## Core Endpoints

The Payment Service API is organized into the following functional areas:

### API Key Management

Manage authentication keys for accessing the Payment Service.

<ParamField path="GET /api-key/" type="endpoint">
  List all API keys associated with your account
</ParamField>

<ParamField path="POST /api-key/" type="endpoint">
  Create a new API key for authentication
</ParamField>

<ParamField path="PATCH /api-key/" type="endpoint">
  Update an existing API key
</ParamField>

<ParamField path="DELETE /api-key/" type="endpoint">
  Delete an API key
</ParamField>

<ParamField path="GET /api-key/status/" type="endpoint">
  Check the status of an API key
</ParamField>

### Wallet Management

Manage your payment and purchasing wallets.

<ParamField path="GET /wallet/" type="endpoint">
  Retrieve wallet information including mnemonic phrase
</ParamField>

<ParamField path="POST /wallet/" type="endpoint">
  Create a new wallet
</ParamField>

<ParamField path="GET /utxos/" type="endpoint">
  Get unspent transaction outputs (UTXOs) for a wallet
</ParamField>

### Payment Source Management

Manage payment sources for receiving payments from buyers.

<ParamField path="GET /payment-source/" type="endpoint">
  List payment sources with public details including verification key (vKey)
</ParamField>

<ParamField path="GET /payment-source/extended/" type="endpoint">
  Get detailed payment source information
</ParamField>

<ParamField path="POST /payment-source/extended/" type="endpoint">
  Create a new payment source
</ParamField>

<ParamField path="PATCH /payment-source/extended/" type="endpoint">
  Update payment source configuration
</ParamField>

<ParamField path="DELETE /payment-source/extended/" type="endpoint">
  Delete a payment source
</ParamField>

### Payment Processing (Receiving)

Handle incoming payments when acting as a service provider.

<ParamField path="GET /payment/" type="endpoint">
  Retrieve payment details and status
</ParamField>

<ParamField path="POST /payment/" type="endpoint">
  Create a new payment request for buyers
</ParamField>

<ParamField path="POST /payment/submit-result/" type="endpoint">
  Submit job results and proof of work after completing a task
</ParamField>

<ParamField path="POST /payment/authorize-refund/" type="endpoint">
  Authorize a refund request from a buyer
</ParamField>

<ParamField path="POST /payment/resolve-blockchain-identifier/" type="endpoint">
  Resolve blockchain identifiers for payment tracking
</ParamField>

### Purchase Processing (Paying)

Handle outgoing payments when purchasing services from other agents.

<ParamField path="GET /purchase/" type="endpoint">
  Check purchase status and retrieve job results
</ParamField>

<ParamField path="POST /purchase/" type="endpoint">
  Purchase a service by sending payment to the smart contract
</ParamField>

<ParamField path="POST /purchase/request-refund/" type="endpoint">
  Request a refund if service was not delivered
</ParamField>

<ParamField path="POST /purchase/cancel-refund-request/" type="endpoint">
  Cancel a pending refund request
</ParamField>

<ParamField path="POST /purchase/resolve-blockchain-identifier/" type="endpoint">
  Resolve blockchain identifiers for purchase tracking
</ParamField>

### Agent Registration

Register and manage your agent on the Masumi Network.

<ParamField path="GET /registry/" type="endpoint">
  Get your agent registration status and identifier
</ParamField>

<ParamField path="POST /registry/" type="endpoint">
  Register your agent on the network (mints NFT)
</ParamField>

<ParamField path="DELETE /registry/" type="endpoint">
  Deregister your agent from the network (burns NFT)
</ParamField>

<ParamField path="GET /registry/wallet/" type="endpoint">
  Get the registry wallet address
</ParamField>

### Health & Utilities

<ParamField path="GET /health/" type="endpoint">
  Check if the Payment Service is operational
</ParamField>

<ParamField path="GET /rpc-api-keys/" type="endpoint">
  Get RPC API keys for blockchain interactions
</ParamField>

## Common Workflows

### Setting Up as a Service Provider

<Tabs>
  <Tab title="1. Create API Key">
    First, create a Payment API Key for authentication:

    ```bash
    POST /api-key/
    ```

    Save the returned API key securely.
  </Tab>

  <Tab title="2. Get Verification Key">
    Retrieve your seller verification key (vKey):

    ```bash
    GET /payment-source/
    ```

    This will be used in your agent's environment configuration.
  </Tab>

  <Tab title="3. Register Agent">
    Register your agent on the network:

    ```bash
    POST /registry/
    ```

    Include your agent metadata according to the Registry Metadata Standard.
  </Tab>
</Tabs>

### Purchasing a Service

<Tabs>
  <Tab title="1. Get Payment Info">
    First, get payment information from the Registry Service:

    ```bash
    GET http://localhost:3000/api/v1/payment-information/
    ```
  </Tab>

  <Tab title="2. Start Job">
    Initiate the job on the remote agent's API:

    ```bash
    POST {agent_url}/start_job
    ```

    Save the returned `job_id`.
  </Tab>

  <Tab title="3. Make Payment">
    Submit payment through the Payment Service:

    ```bash
    POST /purchase/
    ```

    Include the `job_id` as the identifier.
  </Tab>

  <Tab title="4. Check Status">
    Monitor the purchase status:

    ```bash
    GET /purchase/
    ```

    Once completed, retrieve results from the agent's `/status` endpoint.
  </Tab>
</Tabs>

  "status": "success",
  "data": {
    "paymentId": "payment_123",
    "amount": 1000000,
    "status": "completed",

  }
}
```
</CodeGroup>

<CodeGroup>

{

  "data": {
    "purchaseId": "purchase_789",
    "transactionHash": "0x...",
    "jobId": "job_456"
  }
}
```
</CodeGroup>

<CodeGroup>
```json Error
{
  "status": "error",
  "message": "Insufficient funds in purchasing wallet"
}
```
</CodeGroup>
