## Overview

The Masumi Registry Smart Contract enables decentralized agent registration on the Cardano blockchain. It provides a permissionless system for AI agents to register their services, making them discoverable and verifiable within the Masumi Network.

<Note>
  Minting validators can be found in the [validators folder](https://github.com/nftmakerio/masumi-payment-service/tree/main/smart-contracts/registry/validators), and supporting functions in the `lib` folder using `.ak` as a file extension.
</Note>

## Building

Make sure to install Aiken and have it available in your path:

[Install Aiken](https://aiken-lang.org/installation-instructions#installation-instructions)

To generate the smart contracts:

```bash
aiken build
```

## Setup

### Installing Dependencies

Install [Node.js](https://nodejs.org/en/download/package-manager) and dependencies:

```bash
npm install
```

### Generating a Testnet Wallet

Create a wallet for testing:

```bash
npm run generate-wallet
```

The address will be saved in `wallet.addr` and the private key in `wallet.sk`.

<Note>
  Top up test ADA at the [Cardano Faucet](https://docs.cardano.org/cardano-testnets/tools/faucet/)
</Note>

## Running Scripts

### Configuration

Set your Blockfrost API key:

<ParamField path="BLOCKFROST_API_KEY" type="string" required>
  Register at [Blockfrost](https://blockfrost.io/) and get your key for preview or preprod network
</ParamField>

<Warning>
  Use the same testnet consistently (preview or preprod). Cardano has multiple testnets.
</Warning>

### Minting Registry Assets

Mint an example registry asset:

```bash
npm run mint
```

Metadata can be configured in the `mint-example.mjs` file.

### Wallet Defragmentation

Defrag the wallet if needed:

```bash
npm run defrag
```

This creates split UTxOs containing only lovelace for better transaction management.

## Testing

You can add tests in any module using the `test` keyword:

```aiken
test addition() {
  1 + 1 == 2
}
```

Run all tests:

```bash
aiken check
```
